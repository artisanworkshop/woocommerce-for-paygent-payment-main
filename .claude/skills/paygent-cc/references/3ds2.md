# EMV 3Dセキュア 2.0（3DS2）

## 概要

EMV 3Dセキュア（ブラウザベース）。`tds2_check` 設定が `yes` の場合に有効。
決済フローが通常トークン決済と異なり、決済ページ（receipt）へのリダイレクトを経由する。

## フロー

1. `process_payment()` → telegram_kind `031` で与信申込
2. WooCommerceの`woocommerce_receipt_{id}`アクションで3DS2リダイレクトページを表示
3. ユーザーが3DS認証完了
4. `woocommerce_thankyou_{id}` で結果確認（`tds2_status_change()`）

## 主要フック

```php
// 3DS2有効時のフック設定（コンストラクタで）
if ( 'yes' === $this->tds2_check ) {
    add_action( 'password_reset', array( $this, 'jp4wc_password_update' ), 10 );
    add_action( 'woocommerce_receipt_' . $this->id, array( $this, 'paygent_3ds2_redirect_order' ) );
    add_action( 'woocommerce_thankyou_' . $this->id, array( $this, 'tds2_status_change' ) );
}
```

## telegram_kind

| コード | 内容 |
|---|---|
| `031` | 3DS2 与信申込 |
| `094` | 情報照会（3DS2結果確認） |

## 設定項目

| 設定キー | 説明 |
|---|---|
| `tds2_check` | 3DS2有効化（yes/no） |
| `tds2_hashkey` | 3DS2用ハッシュキー |
| `no_tds_card` | 3DS非対応カードを許可するか |
| `attempt` | アテンプト認定対応 |
| `attempt_notice_email` | アテンプト通知メールアドレス |

## tds2_status_change() の処理

サンクスページでの3DS2結果確認。`094`（情報照会）でPaygentから決済ステータスを取得し、
`payment_status`を確認してWooCommerceオーダーステータスを更新する。

## セキュリティ

- `tds2_hashkey` はPaygentから発行される固有キー（公開しない）
- パスワードリセット時にハッシュキー更新が必要（`jp4wc_password_update()`）
- アテンプト認定：3DS非対応カードの決済可否は加盟店のリスク判断に基づく
