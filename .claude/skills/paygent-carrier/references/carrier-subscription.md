# キャリア継続課金

## 概要

WooCommerce Subscriptionsと連携したキャリア決済の定期課金。
`class-wc-gateway-paygent-mb.php` がサブスクリプションフックを処理。

## telegram_kind（継続課金）— PDF仕様書準拠

| コード | 内容 |
|---|---|
| `120` | 継続課金申込 |
| `121` | 継続課金売上要求 |
| `122` | 継続課金取消要求 |
| `124` | 継続課金終了 |
| `125` | 継続課金情報照会 |
| `126` | 継続課金変更 |
| `104` | ユーザ認証要求（共通） |

**注意**: 旧リファレンスにあった 551/561/572 は誤り。正しくは 120/121/122。

## 継続課金申込フロー（120）

```php
// 初回注文時に継続課金申込（120）
// → Paygentが継続課金IDを発行
// → 継続課金IDをオーダーメタ（_paygent_running_id）に保存
// → WooCommerce Subscriptions のcronで毎月121を送信
```

## 継続課金IDの保存

```php
// save_trading_id_running_id() で処理
if ( '120' === $telegram_kind ) {
    // 継続課金申込時は running_id を保存
    $order->update_meta_data( '_paygent_running_id', $response['result_array'][0]['running_id'] );
}
```

## permit_statuses（継続課金）

```php
'sale_cancel' => '122', // 継続課金取消
```

## 契約種別

各キャリア資料に「包括契約」と「直接契約」の2種があり、加盟店の契約形態によって異なる手続きが必要。
